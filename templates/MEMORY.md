# MEMORY.md — Long-Term Operational Memory

_Persistent notes that keep future sessions coherent: stable facts about the human, feedback you've been given about **how to work**, ongoing project context, and pointers to external systems. This whole file is loaded into context every session (via the `@profile/MEMORY.md` import in CLAUDE.md), so **keep every entry terse — a line or two** — and prune what's stale._

_Not the same as the vault (`$VAULT_PATH`): the vault holds knowledge/notes (people, projects, concepts, the daily journal); MEMORY.md holds the meta-knowledge about working with this user and project. And not the same as `USER.md`: USER.md is the full profile of the human — the **User** section here is just high-signal working notes plus a pointer to it._

## User

_Role, expertise, goals, working style — enough to tailor your behaviour. The full profile lives in `USER.md`; keep this short._

## Feedback

_Guidance the user has given about how you should work — corrections **and** confirmed-good approaches. Lead with the rule, then a brief **why** (so you can judge edge cases later)._

## Project

_Ongoing work, decisions, deadlines, incidents — context not derivable from the code or git history. Convert relative dates to absolute ("Thursday" → a real date). Decays fast — prune aggressively._
